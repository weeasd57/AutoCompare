Frontend Experience

What your visitors see when they compare vehicles.

Landing & Search

Neo‑brutalism landing page with powerful search that lets the user pick vehicles using fuzzy search and smart suggestions. Fully responsive and optimized for desktop and mobile.

  • Search by make / model / year
  • Client-side comparison store (Zustand)
  • Instant navigation to comparison view

Comparison View

Side‑by‑side comparison layout with quick stats, spec table and visual charts. Highlighted winners per category help users understand differences in seconds.

  • Up to 4 vehicles at once
  • Performance, fuel and capacity sections
  • Export comparison as PDF report

Dark Mode

Built‑in light / dark theme toggle powered by a small ThemeContext with localStorage persistence. Works on all pages including the admin panel.

  • CSS variables for both themes
  • No layout shift or flash of wrong theme
  • One switch for the whole application

Admin Dashboard

Manage the full vehicle catalog with a clean, focused interface.

Fleet Manager

Table‑based dashboard for all vehicles with image, specs, price and quick actions. Includes bulk selection and delete confirmation modals.

  • Search by make or model
  • CSV export of the current database
  • Image URL list support (multiple images per vehicle)
  • Soft neo‑brutalism UI with bold accents

Add & Edit Forms

Structured two‑column forms for basic and technical specs. Validation is done on the server via Next.js API routes and MySQL.

  • Normalized make / model naming
  • Multiple vehicle images (upload files or import URLs)
  • Primary image extraction + brand‑logo fallback
  • Reusable form layout for add / edit

CSV Import

Import hundreds of vehicles in one go using a simple CSV format. Includes a small preview table and error summary so the admin knows exactly what happened.

  • Flexible column mapping (make, model, year, price, ...)
  • Success / fail counters with messages
  • Perfect for seeding demo data

Ads & Google Image Search API

Optional monetization + optional image sourcing for admin tools.

Google AdSense Ready

The app supports Google AdSense via environment configuration. When enabled, you can render ad slots using the built-in GoogleAdSlot component.

  • Requires: NEXT_PUBLIC_ADSENSE_CLIENT (example: ca-pub-xxxxxxxx)
  • Optional slot envs: NEXT_PUBLIC_ADSENSE_SLOT_HOME, NEXT_PUBLIC_ADSENSE_SLOT_VEHICLES, NEXT_PUBLIC_ADSENSE_SLOT_COMPARE
  • Create ad units in AdSense and use their Slot IDs
  • AdSense script is loaded only when configured

Google Image Search API

Admin tools can fetch candidate vehicle images using Google Custom Search (Image). This is implemented as a server API route.

  • Endpoint: /api/vehicle-image?make=...&model=...&year=...
  • Requires: GOOGLE_API_KEY and GOOGLE_CSE_ID
  • Supports pagination using start parameter

Note: Both integrations require your own Google accounts and IDs (publisher/client ID, ad slot IDs, API keys, and CSE ID). Do not hardcode secrets in the frontend.
